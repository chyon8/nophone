## 프로젝트가 뭔가

위시켓 매니저가 프로젝트 검수·공고 등록을 위해 의뢰인과 통화할 때, **고객 말이 끝나면 AI가 "다음에 그대로 읽을 질문"을 화면에 띄워** 인지 부담을 없애는 통화 코파일럿. 기록·요약은 부산물이지 목적이 아니다. (DESIGN.md "진짜 문제", 2026-07-13 개정)

## 이번 세션에서 있었던 일 (시간순)

1. **Cluely(범용 화상회의 AI 코파일럿)와 비교 검토** — 구조는 비슷하지만(오디오→전사→LLM 실시간 지원, 기록=부산물), 결정적으로 다른 건 ① 온디맨드 Q&A가 아니라 **프로액티브** 질문 제안 ② **위시켓 도메인 특화**(범용 도구가 못 하는 부분) ③ 은폐 기능 불필요(전화라 화면 공유 자체가 없음). 방향은 맞다고 결론.
2. **DESIGN.md "진짜 문제" 개정** — 옛 정의(통화 지식 팀 자산화·기록 중심)를 "질문 제안·인지부담 최소화, 기록=부산물"로 교체. 성공 기준을 **품질 1순위, 속도는 제약(2~3초 바닥선)**으로 확정 — 처음엔 "속도가 핵심 지표"였는데, 느려도 좋은 질문이 빨라도 쓸모없는 질문보다 낫다는 논리로 뒤집음.
3. **`suggest.py` 프로토타입 (v1)** — 화면 없는 터미널 제안 루프. 모델 A/B 테스트: `gpt-4.1`이 속도(첫토큰 ~570ms)·품질(자연스러운 구어체) 둘 다 1위, `gpt-5-mini`는 느리고 괄호 나열식 만연체라 "그대로 읽기" 원칙 위반으로 탈락. PLAN.md V4로 기록.
4. **실제 샘플 통화 전사 → replay 모드로 품질 확인** — 결함 3개 발견: ① 방금 들은 답을 무시하고 직전 질문 반복 ② 이미 물은 걸 또 물음 ③ 한 번에 질문 두 개.
5. **사용자가 `SCORING_SPEC.md` 제시** — 이전에 만들어둔 "12섹션 confidence 채점 → 완성도 계산 → 질문 우선순위 → 공고 draft 출력" 스펙. 이걸 정식 채점 기준으로 채택하기로 결정.
6. **설계: 2루프 구조** — 매 발화마다 12섹션 전체를 분석하면 느려지므로 분리:
   - **빠른 루프**(고객 발화 종료마다, ~0.7초): 섹션 상태 + 최근 대화 원문 → 대사 1개 스트리밍
   - **느린 루프**(백그라운드): 대화록 전체 재채점 → 12섹션 confidence 갱신
   - **통화 전**: 사전 브리핑 1회 채점 (뭐가 비었는지 알고 시작)
   - **통화 후**: 종료 조건(완성도≥70 + 핵심 3섹션≥50) 충족 시 **공고 draft 생성 — 아직 미구현**
   - DESIGN.md에 "제안 엔진" 섹션으로 반영.
7. **구현 (v2)** — `scoring.py` 신규(채점 계층: confidence는 LLM, 합산·우선도·종료 판별은 코드), `suggest.py` 전면 개정(체크리스트를 SCORING_SPEC 12섹션으로 교체, 절차 안내 모드 추가, 직전 제안 반복 방지).
8. **replay로 재검증** — 결함 3개 해소 확인, 속도 평균 725ms 유지, 채점표가 실제 통화 내용과 부합.
9. **커밋·푸시 완료** (`273b998`, `7f3d7f1`).

## 2026-07-21 세션 — 집에서 전화기 없이 검증할 "롤플레이 시뮬 모드" 결정

**맥락**: 다음 우선순위는 "제안 엔진 v2를 실제 통화(반응형 상호작용)로 검증"인데, 집에는 전화기(USB 2채널 하드웨어)가 없다. replay는 고정 대본이라 **고객이 AI 제안에 반응하지 않아** 상호작용 품질의 진짜 시험대가 못 된다는 게 이미 짚은 한계. 그래서 하드웨어 없이 반응형 상대를 만드는 방법을 논의했다.

**검토한 선택지** (고객 역을 누가 하느냐):
- **A. 내가 고객(음성), 시스템이 코파일럿** — 빌드 작음. 단 내가 고객을 연기해 실제 업무 자세와 다름.
- **B. 내가 매니저, AI가 고객** ← **채택**. 코파일럿 제안을 내가 매니저로서 내보내면 AI 고객 시뮬레이터가 페르소나대로 반응. 실제 업무 흐름과 가장 같고, 미검증인 "상호작용 품질"을 정면으로 시험.
- **C. 완전 자동 (AI가 양쪽)** — B가 되면 거의 공짜로 딸려옴(매니저 턴을 직전 제안으로 자동 주입). 무시·재질문·이중질문 회귀 테스트용. B 검증 후 붙임.

**추가 결정 (2026-07-21)**: **음성 안 씀. 매니저 턴은 타이핑.** → 마이크·STT 아예 불필요. 순수 텍스트 대화 루프(= replay의 진화판인데 상대가 반응형 AI 고객).

### B(타이핑) — ✅ 구현·검증 완료 (2026-07-21)

**바로 돌리는 법** (집에서 이것만 치면 됨):
```bash
uv run suggest.py --simulate sample/scenario_example.txt --briefing-file sample/briefing_coreline.txt
```
`나(매니저) >` 프롬프트에 대사를 타이핑 → AI 고객이 반응 → `💬` 다음 매니저 대사 제안 + `📊` 채점. 빈 줄/`/quit` 로 종료. **브리핑은 권장(옵션 아님)** — 없이 시작하면 코파일럿이 프로젝트를 콜드스타트에서 환각한다(실측: 브리핑 없을 때 "전원 연계 방송 플랫폼"으로 지어냄 → 붙이니 사라짐).

**만든 것**: `customer_sim.py`(AI 고객 respond), `suggest.py --simulate` 분기 + `simulate()`, `sample/scenario_example.txt`(합성 시나리오, 새로 만들 땐 복사해 수정), `sample/briefing_coreline.txt`(시나리오와 짝인 등록 정보). sample/은 gitignore라 이 2개는 커밋 안 됨 — 새 환경이면 다시 만들어야 함(내용은 이 문서에 요지 있음, 시나리오=필라테스 3지점 회원권·예약 앱).

**스모크 검증 결과**: 반응형 대화 정상, 답변무시·재질문·이중질문 미발생, 채점 43→54% 순증, 제안 첫글자 평균 ~650ms. 남은 튜닝은 이 모드로 계속.

**참고 — 원래 스펙** (구현 근거):
새 모드 `suggest.py --simulate <시나리오파일> [--briefing-file <브리핑>]`:
1. 시나리오 파일(= AI 고객의 "진실+페르소나")과 브리핑(= 매니저가 통화 전 아는 부분 정보) 로드. 브리핑 있으면 통화 전 1회 채점(기존 그대로).
2. **매니저 턴**: 프롬프트에서 `input()`으로 한 줄 받음 → `transcript`에 `("me", text)` 추가. (마이크·STT 없음)
3. **AI 고객 턴**: `customer_sim.respond(...)` → `("customer", text)` 주입 + 화면 표시.
4. 그 고객 턴이 **기존 `on_event("customer","completed")` 경로를 그대로 트리거** → `💬` 제안 + 백그라운드 `📊` 재채점 (지금 코드 손 안 댐).
5. 2로 반복. 빈 입력 또는 `/quit` → 종료 → `report()`.

- **`customer_sim.py` 신규(작음)**: `async def respond(client, model, scenario, transcript_lines) -> str`. 페르소나 프롬프트 = *IT 프로젝트 의뢰인, `scenario`가 진실. 매니저 말에 자연스럽게 답하되 **가끔 모호하게·되묻고, 정보를 한 번에 다 쏟지 않음**. 1~3문장 구어체. 이미 말한 것 반복 안 함. 절차·정책은 의뢰인답게(모르면 모른다고).* 모델은 gpt-4.1 기본.
- **`sample/scenario_example.txt` 신규**: 합성 시나리오 1개(실제 고객 아님) + 템플릿 주석. sample/은 gitignore라 로컬 전용.
- **안 하는 것(지금)**: 고객 목소리 TTS(고객 대사는 화면으로 읽음, Phase 2 영역), C(완전 자동)는 B 검증 후.

**검증 기준**: 매니저 오프닝 타이핑 → AI 고객이 반응형으로 답 → 코파일럿이 매니저 다음 대사 제안 → 반복하며 채점 오름. **replay에서 안 잡히던 상호작용 결함(답변 무시·재질문·이중질문)이 반응형 상대에서도 안 나오는지** 확인. 이게 청크 D 품질 검증 도구 겸 H1(리허설, 하드웨어 없는 선행분).

**진행 규칙**: 위는 "설계 확정"까지다. 새 세션에서 구현 **시작 전에** PLAN.md 다시 읽고, 사용자에게 "시작해" 명시적 지시 받은 뒤 코드 착수 (CLAUDE.md 룰 7).
